机器学习基石八

上次学到：

learning happens if finite \(d_{vc}\), large N, and low \(E_{in}\)


target function会产生一堆的样本，我们的资料，这些资料从哪里来？除了从target function得到他的y的部分，它的x是由某个我们所不知道的分布产生的，未来在测试的时候，我们也要用一样的分布，我们的演算法从这些资料和一个好的hypothesis set里找到一个好的的hypothesis。好的hypothesis set是vc维是有限的，好的hypothesis是Ein是很小的。
要是资料有噪音呢？

之前信用卡的例子

noise in y: good customer, ‘mislabeled’ as bad? 应该发信用卡，但却记录错误，没有发

noise in y: same customers,different labels? 条件一模一样的顾客，不一样的结果。有一个人可能是专家一号决定的，另一个可能是专家二号决定的。反映到图上就是两个点重叠在一起，但是一个是圈圈，一个是叉叉

noise in x: inaccurate customer information?

在有杂讯的时候，VC bound还能不能起作用？
VC bound的核心：marbles。

只要我们的每个example，来自某个joint distribution，（x,y）在这个joint distribution下独立同分布，那么我们的VC bound还是可以用的。
Target Distribution P(y|x):characterizes behavior of ‘mini-target’ on one x 不是一个目标函数，而是一个目标的分布，对每一个点（每一个弹珠）最理想的预测值是什么。

如上图，圈圈的几率是0.7，叉叉的几率是0.3，大多数人都会猜圈圈，要是圈圈是比较好的选择，那么你犯得30%的错误是怎么回事？这30%其实可以看成在这个最好的选择上面的杂讯。target distribution这个目标的分布实际上告诉我们说最好的预测是什么，然后有多少比例的杂讯。这是一种看法。

goal of learning: predict ideal mini-target (w.r.t P(y|x)) on often-seen inputs(w.r.t P(x))在常见的点上，我们要做的好

我们想要知道的是g和f到底一不一样。
错误测量

之前我们所使用的是Eout来衡量g和f是否一样。更一般的，我们用打分数的方式：给你一个g，来看看g和f到底是否接近。
classification error我们常常把他叫做“0/1”错误。
很多时候，我们所做的错误的衡量，都可以考虑一个点上到底有多少错误，然后再加起来求平均。刚才就是考虑每个点是对还是错，然后求平均，这种方式是pointwise

will mainly consider pointwise err for simplicity

有哪些pointwise的方式

我们对错误的衡量会怎样影响学习？
interplay between noise and error:P(y|x) and err define ideal mini-target f(x)
P(y=1|x)=0.2,P(y=2|x)=0.7,P(y=3|x)=0.1


1.1 = 0.7 x \({(2-1)}^2\) + 0.1 x \({(3-1)}^2\)
什么样的错误衡量，配和上分布，会告诉我们最理想化的那个目标函数长什么样子。上图右边是加权平均值。

错误衡量对演算法来说很重要。

extended VC theory/‘philosophy’ works for most H and err


指纹的情况用在超市上面时，若发生false reject，就是应该给顾客折扣，但是没有给，那么这个顾客很可能就会说他不会再来这个超市，并且叫他的朋友也不要来，对于false accept，给了不该给的顾客折扣，那么今天超市很可能就少赚一点钱。对应于这种情况，超市有自己的一个成本函数表，如上右边的表格

当指纹情况用于CIA情况的时候，false accept的情况是当一个没有权限进入系统时，但是却错误的给了他权限进入，这可能是CIA不能接受的，所以CIA的成本函数和超市的情况又不一样了。如上图右边。而相对于false reject相对于前面的一种情况来说却不是很重要的事。

err is application/user-dependent 不同的情况，用的错误函数也不一样


设计演算法的时候，就要把衡量错误的方式考虑进去，最后想用什么方式衡量，就告诉给你的演算法，但是很多时候，这个并不容易做。要把使用者真正心里想的数字化很困难，所以我们使用替代的方式。

不同的错误有不同的重要性：weighted classification
如何解决这个问题？Ein越小越好

用之前学过的两个演算法尝试去解决

PLA:doesn’t matter if linear separable.:-)
pocket: modify pocket-replacement rule—— if \(w_{t+1}\) reaches smaller \(E_{a}^{b}\) than \(\hat{w}\), replace \(\hat{w}\) by \(w_{t+1}\)


通过复制的方式，使得有权重错误的样本被多次计算

using ‘virtual copying’,weighted pocket algorithm include:

  1. weighted PLA:randomly check -1 example mistakes with 1000 times more probability
  2. weighted pocket replacement: if \(w_{t+1}\) reaches smaller \(E_{a}^{b}\) than \(\hat{w}\), replace \(\hat{w}\) by \(w_{t+1}\)
    systematic rout(called ‘reduction’):can be applied to many other algorithms!
