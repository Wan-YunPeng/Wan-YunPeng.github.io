Kernel Ridge Regression


上次讲到representer theorem，说什么时候我的w可以表示成一堆z向量的线性组合呢，如果你要处理的是一个有regularized的linear model，我们原来是linear model，加上一个L2 regularization后，可以得出你最后最好的那个w是一堆z的线性组合。这是我们上次得到的结果。

我们今天要探讨的是怎么样把我们以前学到的regression变成kernel的形式。
我们之前学的linear regression是试图看看y和预测出来的分数的差值的平方：$$err(y,w^Tz)=(y-w^Tz)^2$$
用这个squared error的方式，想办法把这个error达到最小，如果把他配合上regularization的化叫做ridge regression.那怎么把ridge regression加上kernel？
如果我们要做ridge regression的话，那么我们要解这样的问题：

我有error的部分，有一个regularized的部分，根据representer theorem，我们知道最后w的最佳解可以表示成z的线性组合。一个可能的方式就是既然我知道我最佳解w是z的线性组合，那我把w的形式带入原式，不再求w，而是求线性组合的系数\(\beta\).

可以把他看成一个\(\beta\)的线性模型，前一部分看起来好像和\(\beta\)的复杂度有关，只是前一部分乘上了一个kernel，后一部分则是\(\beta\)作用于kernel，把kernel想象成一种转换出来的资料上，想成\(\beta\)在经过转换之后的资料上的error。可以在化简为另一种形式，只是关于\(\beta\)的。
剩下一个问题：我们要怎么样解出最好的\(\beta\)?

这是我们得到的关于\(\beta\)的最终方程式，可以看成是一个\(\beta\)的二次式，它是一个无条件的最佳化问题，第一步就是算算梯度什么时候会为0.

我们想要梯度为0，其中一个可能的解就是让上式括号的值为0：

但是这个反矩阵是不是一定存在呢？如果你的\(\lambda\)大于0，因为合法的kernel，K一定是半正定的，那么在对角线上再加上一个小小的\(\lambda\)之后一定可以变成一个可逆的矩阵。所以这个解一定存在，那你要怎么算？如果你用一般求逆矩阵的程式去算的话，一般花费\(O(N^3)\)的力气。
那么现在理论上我们可以很轻易的做non-linear regression.以前是你先坐transform，之后在z空间里做regression的动作；现在是另外一种方法，你可以用kernel的方法去解那个潜藏在z空间里面的那个linear regression的解是什么。

Support Vector Regression Primal

least-squares SVM(LSSVM) = kernel ridge regression for classification

如果只看边界的话，几乎没有什么区别，但是你看方框框的点，右边这个几乎每个点都是support vector，为什么会这样？
LSSVM: similar boundary,many more SVs\(\beta\)很多都不为0，这代表未来在做预测的时候会多花很多的时间，也就是你的g会很肥大：slower prediction,dense \(\beta(BIG g)\)
dense \(\beta\): LSSVM,Kernel LogReg; sparse \(\alpha\): standard SVM
所以下一步，我们想要的是\(\beta\)也是sparse，大多数是0
我们先考虑一个很特别的问题：tube regression，意思是这个regression下的分界线有一个中立区，下图蓝色部分，对于中立区中的错误的点，不扣他的分：

相应的错误计算方式在上面列出来了，
接下来就是要把这个L2-regularized tube regression拿去求解得到sparse的\(\beta\)

对比一下tube和squared两幅图，作出他们的错误图：

我们发现向外走的时候红色长得很快，蓝色长得很慢，这样有好处，往外走长得快表示比较容易受到杂讯的影响，如果你现在有一些杂讯的话，很有可能你会为了迁就这些杂讯，为了不犯太大的错误，而影响了你的regression的结果。还有就是tube在比较小的范围内和square是比较相似的。

我们会先模仿之前解SVM的解法，先把他写成一个QP问题，然后透过dual等等方式解他，解完之后我们发现sparse属性。

为了和之前的SVM扯上连接，做了一些小小的改变，第一是把系数换成了1/2和C，然后把w0独立出来变成b。现在我们有了要解的最佳化问题，还要把max这些拆开变成一个二次规划问题。

原来SVM是引入\(\xi_n\)，同样，这里我们也引入\(\xi_n\)，这个变数记录了我们要的和分数的差距在\(\epsilon\)之内，但是要是在\(\epsilon\)之外的话，我们就要收\(\xi_n\)的钱。这个新的变数放到最佳化的问题你，并且是大于等于0的。但这个还不是一个二次规划的问题，因为我们还有一个绝对值在这边，把他拆成两个部分。用两个变数来记录。那我们就得到了一个二次规划的问题。
Support Vector Regression（SVR） primal: minimize regularizer + (upper bube violations \(\xi_n^\wedge\) & lower violations \(\xi_n^\vee\))
