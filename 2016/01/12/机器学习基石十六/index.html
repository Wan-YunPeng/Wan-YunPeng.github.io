Three Learning Principles

上次课讲了一个很重要的工具：validation，我们要怎么做？留下一部分资料作为我们的validation data，验证资料，可以留下一份验证资料或者交叉的方式留下验证资料，目的是希望模拟你在做测试的过程，这样你就知道不同的model表现得好不好。

Occam’s Razor

entities must not be multiplied beyond necessity————William of Occam不要对一个东西多做过多的处理，应该用越简单的方式越好，把那些不必要的东西”剃掉”
‘Occam’s razor‘ for trimming down unnecessary explanation

我们希望不要对资料做过多的解释。这在machine learning里是什么意思呢？他实际上就呼应了我们讲的：对资料最简单的解释也就是最好的解释

The simplest model that fits the data is also the most plausible

就像下面这张图一样：

那么现在有两个问题：

  1. What does it mean for a model to be simple? 什么叫做一个简单的模型
  2. How do we know that simpler is better? 我们为什么知道简单是比较好的

我们过去讲过什么叫做简单的hypothesis，你有单一的hypothesis的时候，我们到底怎么说它是简单的？

一个简单的hypothesis应该是“看起来”很简单，就想刚才那个大大的圆看起来很简单，不像另外一个弯弯曲曲的曲线，“看起来很简单”的意思实际上是我们只需要少数的参数；我们还提到过怎么说一个model是简单的，我们说一个model，有效的hypothesis数量不是很多的话，我们就说他是简单的。这两种解释的方法实际上是有关联性的，如果我们有一个hypothesis set，它最多只有\(2^l\)个hypothesis，那你的一个hypothesis最多需要多少的参数来描述？用bit来表示的话，你有\(2^l\)个hypothesis，那么你只需要l个bits就可以描述\(2^l\)这么多个排列组合

所以如果你今天大大的model的complexity很小，是简单的话，那某种程度也可以表示它里面的hypothesis应该也是简单的。

simple:small hypothesis/model complexity透过简单的hypothesis或model都可以对你的资料进行简单的解释

那为什么简单的就是好的呢？这里提供一个哲学的解释：

假设你有一个简单的model，简单的model就是成长函数很小，长得很慢，这代表什么？如果你今天随便丢一组资料，意思是你有一组输入，输出的部分你就丢铜板来产生，丢铜板产生大家可以想象，应该是没什么规律的，那如果这样烂烂的资料、随机产生的资料丢给你的模型会发生什么事？你就只有很小的机会能找到一个Ein是0的。所以乱乱的资料导致的结果是大部分的时候资料都没有办法分开，那反过来说，如果你今天有一组资料找到一个很完美的模型，可以把这些资料分开，那这是有显著性的，这表示你的资料应该不是乱乱的资料。所以你用简单的模型把资料分开的时候，就可以说资料是具有某种规律性的，相反的，你用很复杂的模型时，你就没办法知道他分开时这个资料到底是乱乱的还是有规律的，因为反正不管什么资料他统统有办法可以分开。

direct action: linear first;
always ask whether data over-modeled 选择模型时，想想这个是不是你现在可以选择的最简单的模型

Sampling Bias

if the data is sampled in a biased way, learning will pordece a similarly biased outcome 如果你今天抽样有偏差的话，那么你学起来也会有偏差，这样的情形我们叫sampling bias

technical explanation :
data from P1(x,y) but test under p2 \(\neq\) p1 : VC fails
philosophical explanation :
study dMath hard but test English: no strong test guarantee学了一学期数学但是期末的时候却考的英语一样

训练和测试要来自同一个分布。

这是林轩田老师当年参加的Netflix的比赛：只要你的推荐系统比他自身的好10%的话，就可以有100万美金。林轩田老师用了validation的方法，显示有13%的提高，但是去没有领到奖，为什么？训练模型时，从手上的资料里随机选出一部份作为他的测试资料，而在实际比赛中却不是这样，netflix是这样选择测试资料的：一个人看了好几部电影之后，我们就把它看到前几部，比如说7部作为训练资料后3部电影当测试资料，也就是说它的训练资料和测试资料是有前后的关系，不是随机取样，而是在时间轴上做了一个前后的取样。因为不是随机取样（validation用的是随机取样）而林轩田老师用了随机取样就使得他的validation的分布和test的分布不太一样。

validation : random examples within D;
test : ‘last’ user records ‘after’ D

那应该怎么办？在实物中我要用前面的资料做训练，后面的资料做测试

practical rule of thumb : match test scenario as much as possible了解你的测试环境，让你的训练环境和测试环境尽可能的接近
e.g. if test : ‘last’ user records ‘after’ D
training : emphasize later examples
validation : use ‘late’ user records

你今天在做训练的时候，因为后面的比较重要，是不是说你应该那些属于时间轴上比较后面的资料我们要加强一下，我们说他的权重比较重或者用一下其他的方式加强，让我整个训练出来的hypothesis会偏向那些时间轴比较后面的资料；又或者说你在做validation的时候，就不要随机选择，而是选择时间轴上比较后面的那些资料来做验证资料。这些方法让你的训练环境和测试环境尽可能的接近，当两者很接近的时候就很有可能让你的学出来的hypothesis在真正测试的时候表现的很好。

last puzzle : danger when learning ‘credit card approval’ with existing bank records ?

第一个个机器学习的问题，是否应该给新顾客信用卡？我们的训练资料是银行已经存在的资料，我们只能知道给了信用卡的人他们有没有把卡刷爆，而对于那些没有给信用卡的人，我们不知道要是我们给了他们信用卡后，他们会不会把卡刷爆，也就是说我们的训练资料实际上是经过筛选的资料，经过们原来的系统筛选说这些人应该要给他信用卡，给了之后我们看他们有没有乱花钱/刷爆卡，但是我们没办法说被我们排除的那些人到底会发生什么事。所以这个资料训练出来的模型用在新的顾客上，我可能接受他，我可能拒绝他，这两个的分布就不一定会一样。需要做一些调整。

Data Snooping

不要”偷看资料”

if a data set has affected any step in the learning porcess,its ability to assess the outcome has been comromised

假设今天有8年的货币交易的资料，我们想从这些资料中导出来看看能不能预测货币的走势，我们拿前6年做训练的工作，拿后两年做测试的工作。

蓝色只放缩前6年，而红色是放缩了8年的资料，相当于测试资料已经影响了我们的训练资料。

snooping : shift-scale all values by training + testing
no snooping : shift-scale all values by training only

怎么处理”偷看”资料这件事？这件事非常难

one secret to winning KDDCups :
careful lalance between data-driven modeling(snooping) and validation(no-snooping)
